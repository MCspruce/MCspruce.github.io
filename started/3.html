认识JSON-UI

阅读进度 -

.bb什么是JSON-UI？..bb

在Minecraft Bedrock Edition（即《我的世界》基岩版，以下简称MCBE）中的用户界面（以下简称UI）有两种： .hh一个是JSON UI，另外一个是Ore UI .hh本文档主要讲述JSON-UI。 .hh.hhJSON-UI就是由JSON文件构成的UI界面，它能允许我们通过修改JSON文件来改变UI，以及在一定的程度内为UI添加一些行为和功能。 .hh首先，JSON-UI的JSON文件主要存放在RP/ui/文件夹内

.bbui文件夹内有什么文件？..bb

_global_variables.json | 全局变量，以后会详细讲解 .hh_ui_defs.json | 声明JSON-UI文件，以后会详细讲解 .hhui_common.json | UI模板，存放常用的控件模板 .hhxxx_screen.json | 某个屏幕文件，存放该屏幕的控件

接下来，详细解读xxx_screen.json的内容

RP/ui/start_screen.json { "namespace": "start", "label": { "type": "label", "text": "Hello World" } } 其中，"namespace"是这个UI文件的标识符。 hh我们通常称它为.bb命名空间..bb hh它必须是唯一的名称，一般用于跨文件引用控件。

文中的"label": {}是.bb控件..bb，你可以把它当作屏幕上某个元素。 .hh.hh而控件开头的 "label" 是.bb控件名..bb，用于在其他控件引用这个控件的元素。 .hh每个控件的控件名必须是唯一的名称。 .hh.hh"text": "Hello World" 就是这个控件中的某个.bb元素..bb，用于完善控件的外观、位置。 .hh.hh整理一下，一个JSON-UI文件的结构是： .hh命名空间 + （控件 -> {控件名 + 元素}）

.bb什么是Ore UI？..bb

上面提到了ore UI，现在简单了解一下。 .hh.hhOre UI就是由Web前端技术（Html5 + CSS + JavaScript）构成的UI界面，与JSON-UI不同的是，它可以运行JavaScript代码！ .hh.hh首先，将Ore UI与JSON-UI对比可以发现： .hh* JSON-UI更利于阅读、修改。 .hh* Ore UI只能通过修改安装包实现美化，而且代码似乎被混淆了，修改难度大大上升。 .hh.hh毕竟Ore UI与渲染龙一样，目前还不能用资源包调用修改它们！
