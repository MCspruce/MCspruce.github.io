要了解json-UI，那就必须了解json(JavaScript Object Notation)语言
java≠JavaScript≠JSON


如果想深入了解JSON，可以前往www.w3school.com.cn/js/js_json_syntax.asp


JSON的文件后缀名为.json
json语言的标识符号必须为英文字符


json文件开头和结尾都必须有{}包括着
json元素是由键和值构成的


键是由双引号""且在双引号里面填入字符串
例如："name"


键后跟着冒号和值
例如：
{"name":"value"}
文中的"value"就是值


在JSON中，值必须是以下数据类型之一
提示：
字符串的前后需要加上""

字符串 示例: "test"

数字 示例: 1 -1 1.1

对象 示例: {"to": "test"}

数组 示例: ["test",1,{"to": "test"}]

布尔值 示例: true false

无值 示例: null
下面详细讲一下值

字符串
"text":"文字"
需要双引号""包围，里面可以是任何内容
提示：
字符串内部的换行最好使用\n


数字
"number":1.0
数字可以为浮点数或整数

对象
"object":{"key":"a","key_a":1}
里面包含许多键和值
但是要注意，里面的键名不能重复
里面的每个键和值都需要逗号分隔

数组
"array":["a",1,{"a":{}}]
里面包含了许多值，但不能包含键名
里面的每个键和值都需要逗号分隔

布尔值
"boolean_value":true
可以确定该键是否为真或假
布尔值可以为true和false
其中
true = 真
false = 假

无值
"null_value": null
空值，代表该值没有任何数据

JSON特点
* 兼容强，每个编程语言都支持json读取，存储，解析
* 比xml文件体积更小
* 易学易用
* 解析速度快


JSON压缩和格式化
> 无论是压缩还是格式化，只要语法正确都能解析
json压缩后文件体积更小，例如 
{"a":"b"}
但是数据较多时很难阅读，格式化更能更好编辑json，例如 
{
  "a":"b"
}
但是这样做会增加文本字节

JSON注释
一般所有编程语言都可以注释代码
json语言的注释为段落注释和单行注释

单行注释的符号是//，//后面的内容在这行不会被解析
段落注释的符号分别为/*和*/，/*后面的内容将不会被解析，直到*/
例如 
{
  //这行不会被解析
  "a": "b"
  /*
  这里不能被解析
  */
}

JSON转义
值为字符串时
不能直接把"填入字符串
只要在"前面加上\就行了
例如：\"

Unicode
JSON的键和值都是支持Unicode的
例如：\U2192
同时Unicode也可以被转义
在Unicode前面多加个\就行了
