面板控件


面板控件，在HTML网页中是一个必不可少的元素，
它可以改变部分子元素的布局！

当然，在JSON UI中也可以改变页面的布局。

接下来，我们会讲以下关于面板的控件：
  panel（面板）
  stack_panel（堆叠面板）
  input_panel（输入面板）
  滚动面板

提示：
size等常用属性被整理至另外一个文件

点击查看

controls

controls属性，我们常常用于在当前控件添加子控件
所有控件都可以使用controls属性

例如

"image": { "type": "image", "texture": "textures/item/apple", "controls": [ { "text": { "type": "label", "text": "test" } }, {"empty@common.empty_panel": {}} ] } 在controls属性中，子控件首尾都有{ }包裹，
由上可知，text,empty是子控件，image是text,empty的父控件。

父控件能改变子控件的锚点、大小范围，这都取决于父控件的大小；
在controls属性中，你可以添加无数个子控件，
子控件中，可以继续使用controls为当前的子控件添加子控件。

面板

在JSON UI中，有一个叫面板(panel)的控件，TA与按钮、图片不同，它是一种无色无味的透明控件。
虽然控件看不见，但它在每个JSON UI必不可缺，起着重大作用。

面板，用于统一管理所有子控件的大致属性（不是改变子控件属性），
例如改变面板控件的layer，该面板控件的子控件layer也在增加。
面板相当于HTML的"div"

例如：

"panel": { "type": "panel", "size": [ 100, 100 ], "controls": [ { "label": { "type": "label", "text": "hello world", "anchor_from": "top_right", "anchor_to": "top_right" } }, { "board": { "type": "image", "texture": "textures/ui/focus_border_white", "ninelice_size": 1, "size": [ "100%", "100%" ] } } ] } 效果如上图所示，白色边框是面板控件的大小；
面板可以改变 "label" 控件的锚点定位，因为锚点是基于父控件定位的。

对于面板控件，没有独有的特殊属性。所以面板控件就讲到这里了。

堆叠面板

堆叠面板控件与普通的面板一样，也是看不见的。

堆叠面板能根据子控件的 "size" 属性堆叠子控件
例如：

"panel": { "type": "panel", "size": [ 100, 100 ], "controls": [ { "stack_panel": { "type": "stack_panel", "orientation": "horizontal", "size": [ "100%", "100%" ], "controls": [ { "label": { "type": "label", "text": "hello" } }, {"empty@common.empty_panel": {"size": [2,2]}}, { "label2": { "type": "label", "text": "world" } } ] } }, { "board": { "type": "image", "texture": "textures/ui/focus_border_white", "ninelice_size": 1, "size": [ "100%", "100%" ] } } ] }

在代码中“hello world”被拆成两个控件，图中的堆叠面板把它们堆叠一起。
所以看起来只是一个控件

你知道吗？
写UI时通常引用common.empty_panel作为间隙

但是，在堆叠面板中。子控件不能改变自身的位置。（改变锚点以及偏移）
因为子控件锚点基于父控件的左上方作为原点并排列堆叠，所以子控件总是在父控件的左上方。
如需修改，需要在子控件内在加上子控件
举个例子：

"stack_panel": { "type": "stack_panel", "controls": [ { "panel": { "type": "panel", // 无效的offset "offset": [ 1, 1 ], "controls": [ { "label": { "type": "label", // 有效的offset "offset": [ 16, 0 ], "text": "hello world" } } ] } } ] }

"orientation"用于改变堆叠面板的堆叠方向 orientation 堆叠方式

值：字符串/列举
默认值：vertical
控制子控件堆叠方向。
vertical - 垂直
horizontal - 水平
在"size"属性中，可以使用"fill"填充堆叠面板剩余空间。

"panel": { "type": "panel", "size": [ 100, 100 ], "controls": [ { "stack_panel": { "type": "stack_panel", "orientation": "horizontal", "size": [ "100%", "100%" ], "controls": [ { "img": { "type": "image", "size": [ "10%", 16 ], "texture": "textures/items/apple" } }, { "img2": { "type": "image", "size": [ "fill", 16 ], "texture": "textures/items/apple" } } ] } }, { "board": { "type": "image", "texture": "textures/ui/focus_border_white", "ninelice_size": 1, "size": [ "100%", "100%" ] } } ] }

这里不在展示"fill"大小效果，有兴趣可以自己尝试！
在堆叠面板中的除 img 外，X轴剩余大小为 90%；
所以img2子控件大小为[ "90%", 16 ]。

操作面板

什么是操作面板？
操作面板可以说是一个普通的面板控件。不像堆叠面板，无法直接改变子控件位置。

那操作面板有什么特别之处呢？
答：操作面板能控制以及限制操作方式，使得面板控件更有互动性。

例如：

"input_panel": { "type": "input_panel", "size": [ 100, 100 ], "controls": [ { "label": { "type": "label", "offset": [ 3, 3 ], "text": "hello world" } }, { "board": { "type": "image", "texture": "textures/ui/focus_border_white", "ninelice_size": 1, "size": [ "100%", "100%" ] } } ] }

效果与普通的面板控件一致，不再展示。

对于操作面板，它的属性你知道多少？

提示
button_mappings 按钮映射已在按钮控件（点击跳转）讲到，这里不再介绍。 modal 启用模态

值：布尔值
默认值：false
值为 true 时，其他控件（除该操作面板及其操作面板子控件）阻止操作（点击、悬停、焦点操作）。

inline_modal 启用线型模态

值：布尔值
默认值：false
值为 true 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止操作（点击、悬停、焦点操作）。

prevent_touch_input 阻止触摸输入

值：布尔值
默认值：false
值为 true 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止触屏操作。

always_handle_pointer 处理指针操作

值：布尔值
默认值：true
值为 false 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止鼠标操作。

always_listen_to_input 总是监听操作

值：布尔值
默认值：true
值为 false 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止操作（点击、悬停、焦点操作）。

[与 inline_modal 属性相反]

always_handle_controller_direction 处理手柄操作

值：布尔值
默认值：false
值为 false 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止手柄操作。

hover_enabled 启用悬停

值：布尔值
默认值：true
未知，用于base_screen的子控件

consume_event 启用事件操作

值：布尔值
默认值：true
值为 false 时，该操作面板所在范围的控件（除该操作面板及其操作面板子控件）阻止操作（点击、悬停、焦点操作）。

consume_hover_events 启用悬停操作

值：布尔值
默认值：true
未知

gesture_tracking_button 手势跟踪按钮

值：字符串
默认值：none
目前仅允许button.true_doll，用于旋转纸娃娃

操作面板就讲这么多了，接下来我们来认识滚动面板

滚动面板

滚动面板是由多个控件组成的面板
因此滚动面板不能是一个单独的控件，不能像上面的面板直接使用 controls 加入子控件
所以使用它需要引用 ui_common.json 文件的 scrolling_panel 控件
即 @common.scrolling_panel

首先展示下滚动面板的代码

"scrolling_panel@common.scrolling_panel": { "$show_background": false,// 显示背景 "$scrolling_content": "stack_panel",// 滚动面板内容 "size": [ "100%", "100%" ]// 滚动面板大小 }, "stack_panel": { "type": "stack_panel", "size": [ "100%", "100%c" ], "controls": [ {"label_1":{"type":"label","text":"hello world"}}, {"label_2":{"type":"label","text":"hello world"}} ] }

接下来，我们先来讲下滚动面板的基本属性

基础属性

$scrolling_content 面板内容

值：字符串/控件
滚动面板的内容，例如上文的代码中的"stack_panel"
目前仅支持上下滚动内容

$show_background 面板背景

值：布尔值
默认值：true
建议为 false 隐藏背景。
值为 ture 时，显示背景。主要用于 pocket UI 档案的可交互屏幕
如需修改背景，请另见其下。

修改滚动面板背景

在滚动面板中，默认的背景可能会不符合你的 UI 风格
那么要如何修改背景呢？ 提示
需要 $show_background 为 true 显示背景，所做的背景才得以显示 "scrolling_panel@common.scrolling_panel": { "$show_background": false,// 显示背景 "$scrolling_content": "stack_panel",// 滚动面板内容 "size": [ "100%", "100%" ],// 滚动面板大小 "$scroll_background_image_control": "hud.scroll_bg" }, "scroll_bg": { "type": "image", "texture": "textures/ui/lava", "fill": true }, "stack_panel": { "type": "stack_panel", "size": [ "100%", "100%c" ], "controls": [ {"label_1":{"type":"label","text":"hello world"}}, {"label_2":{"type":"label","text":"hello world"}} ] } 像上面，使用 $scroll_background_image_control 更改背景 $scroll_background_image_control 面板背景

值：字符串/控件
默认值：common.scroll_indent_image
更改滚动面板的背景，背景不会随面板的滚动而滚动

提示
引用控件时，切记要在值的开头加上该文件的命名空间
例如在 "hud_screen.json" 的控件，命名空间就是 "hud."
在引用时就是 "hud.某个控件" 改变背景在滚动面板内大小你还需要使用此属性 $background_offset 面板背景大小

值：字符串/大小
默认值：[ "100%", "100%" ]
更改滚动面板的背景大小

改变背景在滚动面板内偏移你还需要使用此属性 $background_offset 面板背景偏移

值：字符串/偏移
默认值：[ 0, 0 ]
更改滚动面板的背景偏移

修改滚动条样式

如果，你想要改变原版的滚动条。 那么接下来，我们将简单地讲下改变滚动条样式！ 当然，不想要显示滚动条可以使用 $scroll_box_visible 。 $scroll_box_visible 滚动条是否可见

值：布尔值
默认值：true
值为 false 时不可见

例如这样更改滚动条
只要加上这两个属性并引用控件即可: $scroll_box_mouse_image_control $scroll_box_touch_image_control "scrolling_panel@common.scrolling_panel": { "$show_background": false,// 显示背景 "$scrolling_content": "stack_panel",// 滚动面板内容 "size": [ "100%", "100%" ],// 滚动面板大小 "$scroll_box_mouse_image_control": "hud.scroll_box_image" }, "scroll_box_image": { "type": "image", "texture": "textures/ui/button_borderless_lighthover", "fill": true, "alpha": "@common.anim_scrollbar_box_fadeout" }, "stack_panel": { "type": "stack_panel", "size": [ "100%", "100%c" ], "controls": [ {"label_1":{"type":"label","text":"hello world"}}, {"label_2":{"type":"label","text":"hello world"}} ] } 提示
引用控件时，切记要在值的开头加上该文件的命名空间！
例如在 "hud_screen.json" 的控件，命名空间就是 "hud."。
在引用时就是 "hud.某个控件"。 在代码中的 "alpha": "@common.anim_scrollbar_box_fadeout" ，就是让滚动条在滚动面板静止不动的状态下隐藏，建议加上去。 滚动面板就讲到这里了！还有很多属性没讲，其中这里只挑了重要的来讲。
