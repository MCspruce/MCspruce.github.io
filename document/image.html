图片的控件类型为image

"type": "image"
以下是图片一些常用属性：

.ep.bb texture ..bb .et图片的路径..et

值：字符串/路径 .hh* 值为空时图片以紫黑材质显示(代表该图片不存在或格式无法解析) .hh例如：textures/ui/anvil_icon .hh根目录以RP（资源包）文件夹开始 .hh图片格式建议为.jpg .png .tga

..ep .ep.bb alpha ..bb .et图片透明度..et

值：浮点数 .hh默认值：1 .hh改变图片的透明度，但不改变图片内的控件透明度

值的范围为0 ~ 1

..ep
其他属性

以下是附属的图片属性，一般用来完善图片控件

tiled 平铺贴图

值：布尔值 | 字符串
默认值：false

值为布尔值时
是否复制多个贴图并平铺至当前图片控件的大小

值为字符串时
字符串只能为"x"和"y"
平铺的方向可以是x轴或y轴方向平铺

tiled_scale 平铺比例

值：数组
例如：[32,32]
如果原贴图大小为16x，将平铺比例设置为32x。平铺贴图时每张贴图大小为32x，而不是16x
使用该属性必须设tiled的值为true

grayscale 黑白滤镜

值：布尔值
默认值：false
是否让图片以黑白方式显示

bilinear 使贴图双线性滤波

值：布尔值
默认值：false
是否让图片以双线性滤波显示

fill 填充贴图

值：布尔值
默认值：false
是否填充图片直到当前控件的大小

keep_ratio 保持贴图比例

值：布尔值
默认值：false
是否让图片保持原有的比例

.ep.bb allow_debug_missing_texture ..bb .et 显示错误材质 ..et

值：布尔值 .hh默认值：true .hh当"textures"使用的路径不存在或图片不存在时，是否使用missing_texture.png

..ep

nineslice_size 伸缩贴图

值：浮点数 | 数组
默认值：0
改变图片的伸缩比例
作为浮点数时必须为正有理数

它有有三种格式

  • 浮点数
  • [x,y]
  • [x¹,y¹,x²,y²]

    • 当不使用nineslice_size时：
    图片会以1:1的比例伸缩
    当值为浮点数时：
    图片伸缩比例等于当前控件比例（会改变图片显示的大小），将外围xxx格像素缩小
    当值为[x,y]时：
    图片伸缩比例等于当前控件比例（会改变图片显示的大小），将外围低于[ x, y ]的像素缩小
    例如：[20,12]
    首先图片的伸缩比例等于当前控件比例
    X轴0~20格像素，Y轴0~12格像素进行缩小处理
    当值为[x¹,y¹,x²,y²]时：
    图片伸缩比例等于当前控件比例（会改变图片显示的大小），将外围低于[ x¹, y¹, x², y² ]的像素缩小
    例如：[2,7,5,9]（设图片大小为32,32）
    首先图片的伸缩比例等于当前控件比例
    第一个伸缩点为左上角的[0,0]到右下方的[2,7]（在这范围的像素会缩小）
    第二个拉伸点为右下角[0,0]到左上方[5,9]（在这范围的像素会缩小）

    uv 图片显示的原点

    值：数组
    默认值：[ 0, 0 ]
    默认的图片原点默认为左上角的[0,0]位置
    建议配合下面的uv_size使用

    例如："uv": [ x, y ]
    参考坐标图
    .tip 注意

    如果第一个值超过贴图的长，那么图片将不会显示
    如果第二个值超过贴图的宽，那么图片将不会显示
    ..tip 如果：
    texture指定的图片大小是2900*1080
    uv是[ 1, 3 ]
    将截取从贴图左上角向右1格，向下3格的像素至右下角边缘的像素作为显示的目标

    uv_size 图片最大显示区域

    值：数组
    用于改变图片的显示区域
    方向如上图所示

    如果：
    texture指定的图片大小是2900*1080
    uv_size是[ 2800, 1080 ]
    会从贴图左上角开始向右2800格，向下1080格的像素至原点的区域为显示的目标
    [原点可通过uv改变]

    force_texture_reload 更改路径时是否重新加载图片

    值：布尔值
    默认值：false
    用于重新加载图片

    .ep.bb zip_folder ..bb .et zip路径 ..et

    值：字符串/路径 .hh与texture属性一致 ..ep .ep.bb texture_file_system ..bb .et 系统路径 ..et

    值：字符串/路径 .hh与texture属性一致，但路径指的是系统路径（会因操作系统而异） ..ep
    动图属性 .hr .tip 提示

    如果第二个值超过贴图的宽，那么图片将不会显示
    ..tip

    那么如何制作动图？

    1.首先，图片如果是Gif图片需要折分并转换为其它图片格式

    2.然后再将每张图片合并为一张图片（水平拼接），例如下方图片

    .tip 注意
    每张图片的长度和宽度要相等
    ..tip

    将图片存储到资源包内的一个位置

    3.然后回到你要添加动图的文件，并添加基础的图片控件和属性
    * 下面以下文代码作演示(texture的值填你刚存放图片的资源包路径)

    RP/ui/start_screen.json "test_image": { "type":"image", "texture":"textures/ui/auto_save", "size":[16,16] }

    4.在控件内部添加变量

    disable_anim_fast_forward 禁用动画快进

    值：布尔值
    默认值：false
    一般使用动图时用true

    uv_size 贴图最大显示区域

    上面讲过了,但要在这强调一次
    作为动图时，uv_size表示的是一帧图片的大小
    如果是128*16的图片，拼接的图片有8张
    那么uv_size就是[ 长÷拼接图片数量, 图片的宽 ]

    填完后…

    ＊不要照抄，这里仅做示范，可能会与你的代码不同

    RP/ui/start_screen.json "auto_save": { "type": "image", "texture": "textures/ui/auto_save", "size": [ 16, 16 ], "uv_size": [ 18, 30 ], "disable_anim_fast_forward": true } 并没有这么简单，上面只是一个静态图片控件

    5.写个新的动画控件

    在动画控件填入

    RP/ui/start_screen.json "image_anim": { "anim_type": "flip_book", "easing": "linear" } 注意：动画控件无需"type"键值，并且动画控件最好与图片控件在同一文件中

    6.在动画控件添加变量

    initial_uv 动图播放的起点

    值：数组
    默认值：[ 0, 0 ]
    值跟上面uv一样

    easing 动画样式

    值：字符串/列举
    以后讲控件动画会详细讲到
    作动图时建议为"easing": "linear"

    reversible 重复播放

    值：布尔值
    默认值：false
    为true时，动图将重复播放

    frame_count 图片数量（帧数数量）

    值：整数
    默认值：1
    拼接的图片有多少张就填多少

    fps 帧数

    值：浮点数
    默认值：1
    数字越大，播放越快

    frame_step 单张图片的长度

    值：整数
    默认值：1
    拼接的每张图片有多长，整数就填多少

    动画控件的属性就这么多了

    填完的效果可能这样的

    RP/ui/start_screen.json "image_anim": { "anim_type": "flip_book", "initial_uv": [ 0, 0 ], "frame_count": 9, "frame_step": 18, "fps": 10, "reversible": true, "easing": "linear" } 7.回到图片控件添加uv变量 例如：

    RP/ui/start_screen.json "auto_save": { "type": "image", "texture": "textures/ui/auto_save", "size": [ 16, 16 ], "uv_size": [ 18, 30 ], "uv": "@image_anim", "disable_anim_fast_forward": true } 在这里，uv的值是字符串
    并且值必须是"@动画控件名称"

    8.检查代码格式是否正确

    好了，动图控件已经完成，记得引用控件才能显示在屏幕上哦
    下面是完整的示例代码

    RP/ui/start_screen.json "image_anim": { "anim_type": "flip_book", "initial_uv": [ 0, 0 ], "frame_count": 9, "frame_step": 18, "fps": 10, "reversible": true, "easing": "linear" }, "test_image": { "type": "image", "texture": "textures/ui/auto_save", "size": [ 18, 30 ], "uv_size": [ 18, 30 ], "uv": "@image_anim", "disable_anim_fast_forward": true }
