什么是按钮？
它能让你打开其他的界面，还会根据四种状态默认，悬停，按下，锁定来改变按钮本身的纹理

你可以@common.button来引用原版的按钮控件

也可以使用type定义button控件

RP/ui/start_screen.json "button": { "type": "button" }

定义按钮属性

提示：
size等常用属性被整理至另外一个文件

点击查看

这些属性可以不用写，因为@commom.button时就已经默认了

default_control 默认状态下子项目的名称

值：字符串/控件名
建议不要更改该属性！
在改变按钮外观时，同时更改control/default控件名有效
引用@common.button时默认为default

hover_control 悬停状态下子项目的名称

值：字符串/控件名
建议不要更改该属性！
在改变按钮外观时，同时更改control/hover控件名有效
引用@common.button时默认为hover

pressed_control 按下状态下子项目的名称

值：字符串/控件名
建议不要更改该属性！
在改变按钮外观时，同时更改control/pressed控件名有效
引用@common.button时默认为pressed

locked_control 锁定状态下子项目的名称

值：字符串/控件名
建议将值设为"locked"！
在改变按钮外观时，同时更改control/locked控件名有效
在引用@common.button时没有声明控件名！

sound_name 按下后播放的音效

值：字符串/音效ID
可用于：buttontoggle可互动控件···
值必须使用RP/sounds/sound_definitions.json的音效ID
你也可以在此文件创建一个新的音效
引用@common.button时默认为random.cilck

sound_volume 按下后播放的音效的音量

值：浮点数
可用于：buttontoggle可互动控件···
值为'0'时系统会判定这是个无音效的按钮
音量会受设备的音量设置所影响
引用@common.button时默认为1.0

sound_pitch 按下时播放的音效的速度

值：浮点数
可用于：buttontoggle可互动控件···
建议值大于'0'
数值越大，播放速度越快！
例如：值为'2'时，播放的倍速就是2.0X
引用@common.button时默认为1.0


按钮映射

直接添加按钮只会改变自身材质，如果需要按钮有实际作用，
需要使用 button_mappings 把键盘/手柄/其他操作方式的输入操作(如button.menu_select、button.menu_cancel)映射转换为游戏内的操作(如button.menu_settings)或用于改变动画状态(详见动画控件)

触发映射条件时而且from_button_id指定按键按下并松手仅触发一次to_button_id事件
下面我们来认识一下如何添加按钮映射吧！

button_mappings 按钮映射组

值：数组
可用于：buttoninput_paneltoggle可互动控件···
用于定义按钮按下时触发的事件

RP/ui/start_screen.json // 示例 "button_mappings": [ { "from_button_id": "button.menu_select", "to_button_id": "$pressed_button_name", "mapping_type": "pressed", "input_mode_condition": "gamepad_and_not_gaze" } ]

button_mappings 子属性
属性名值介绍
from_button_id字符串触发条件的按钮ID
to_button_id字符串执行操作的按钮ID
mapping_type字符串触发条件
（可能的值：下面有说明）
input_mode_condition字符串输入模式条件
（可能的值：下面有说明）
scope字符串范围
（可能的值：view,controller）
handle_select布尔值条件：手柄选择
handle_deselect布尔值条件：手柄不选择

> 当from_button_id触发时会将from_button_id转换为to_button_id
> to_button_id事件可当作动画的开始，重置，结束事件
> mapping_type值为global时，触发form_button_id会优先触发to_button_id事件
> 按钮映射的事件是单次执行的

触发条件：

  • pressed - 按下时
  • focused - 聚焦时
  • double_pressed - 双击时
  • global - 始终（触发from_button_id后）

输入模式条件：

  • not_gaze - 无条件
  • gamepad_and_not_gaze - 使用游戏手柄

以下是一些常用的button_id：

  • button.menu_cancel（Esc键或返回键）
  • button.exit（退出屏幕）
  • button.menu_ok（Enter键）
  • button.menu_select（单击或点击）
  • button.hide_hud（F1键)
  • button.hide_paperdoll（纸娃娃）
  • button.menu_up ↑
  • button.menu_down ↓
  • button.menu_left ←
  • button.menu_right →
  • button.menu_autocomplete（Tab键）
  • button.menu_autocomplete_back（上个Tab键）
  • button.menu_tab_left（左Tab键）
  • button.menu_tab_right（右Tab键）

如果仅需按下按钮触发映射事件，可以先引用"commom.button"，然后使用$pressed_button_name .hh提示：$pressed_button_name的值等于to_button_id

更改按钮外观

接下来，我们将外观分成几部分讲解

文字按钮

类似于start界面中间的三个按钮，这种按钮通常叫做文字按钮
首先举个例子：

RP/ui/start_screen.json "play_button@common_button.light_text_button": { "size": ["100%","100%"], "$pressed_button_name": "$play_button_target", "$button_text": "menu.play" }

属性名值介绍
$pressed_button_name字符串值等于上面的to_button_id（按下就可以触发这个按钮事件）
$button_text字符串按钮显示的文字（能被.lang文件翻译）
$border_visible布尔值显示按钮边框
$button_image字符串/控件按钮外观
$button_image_visible布尔值显示按钮外观
$button_image_size数组/大小按钮外观大小
$image_offset数组/偏移按钮外观偏移
$button_image_fill布尔值按钮外观填充
$image_anchor字符串/列举按钮外观偏移点

tip：
@common_button.light_text_button改为@common_button.dark_text_button以黑暗外观呈现（没改$button_image情况下）

图片按钮

类似于start界面左下角的按钮，这种按钮通常叫做图片按钮
首先举个例子：

RP/ui/start_screen.json "icon": { "type": "image", "texture": "textures/ui/achievements", "layer": 3, "size": [16,16] }, "button@common_button.light_content_button": { "size": ["100%","100%"], "$button_content": "start.icon", "$pressed_button_name": "button.menu_achievements" }
属性名 值 介绍
$pressed_button_name 字符串 值等于上面的to_button_id（按下就可以触发这个按钮事件）
$button_content 字符串/控件 按钮主要内容
$border_visible 布尔值 显示按钮边框
$button_image 字符串/控件 按钮外观
$button_image_visible 布尔值 显示按钮外观
$button_image_size 数组/大小 按钮外观大小
$image_offset 数组/偏移 按钮外观偏移
$button_image_fill 布尔值 按钮外观填充
$image_anchor 字符串/列举 按钮外观偏移点

tip：
> $button_content的值为命名空间.控件名，用于引用控件在按钮内部显示
> @common_button.light_content_button改为@common_button.dark_content_button以黑暗外观呈现（没改$button_image情况下）

无背景内容按钮

上方演示的按钮都是有背景的，那么如何仅显示内容呢？
首先举个例子：

RP/ui/start_screen.json "icon": { "type": "image", "texture": "textures/ui/achievements", "layer": 3, "size": [16,16] }, "button@common_button.no_background_content_button": { "size": ["100%","100%"], "$button_content": "start.icon", "$pressed_button_name": "button.menu_achievements" }

$pressed_button_name 按钮映射事件

值：字符串
值等于上面的to_button_id（按下就可以触发这个按钮事件）

$button_content 按钮主要内容

值：字符串/控件
值为"命名空间.控件名"，用于引用控件在按钮内部显示

个性化按钮

这种按钮通常用于定义新的外观
首先举个例子：

RP/ui/start_screen.json "icon": { "type": "image", "texture": "textures/ui/achievements", "layer": 3, "size": [16,16], "alpha": "$icon_alpha", "$icon_alpha|default": 1 }, "test_button@common.button": { "size": ["100%","100%"], "$pressed_button_name": "button.menu_achievements", "controls": [ // 当前显示的内容取决于按钮状态 // { // 默认状态 "default@icon": {} }, { // 悬停状态 "hover@icon": {"$icon_alpha"： 0.8}// 当按钮为悬停状态时，内容透明度为0.8 }, { // 按下状态 "pressed@icon": {} }, { // 锁定状态 "locked@icon": {} } ] }

$pressed_button_name 按钮映射事件

值：字符串/映射事件
值等于上面的to_button_id（按下就可以触发这个按钮事件）


tip：
> controls的子控件名取决于
default_control,hover_control,pressed_control,
locked_control定义的控件名，locked_control状态需要自行定义为locked。
> 注意：按钮无锁定状态可以不用locked子控件

关闭按钮

这种按钮通常用于关闭当前屏幕

RP/ui/start_screen.json "close_button@common.close_button": { "size": ["100%","100%"], "$close_button_to_button_id": "button.menu_exit" }

$close_button_to_button_id 按钮映射事件

值：字符串/映射事件
值等于上面的to_button_id（按下就可以触发这个按钮事件）

修改外观与个性化按钮类似，这里不多一一介绍

返回按钮

这种按钮通常用于返回上个屏幕

RP/ui/start_screen.json "back_button@common.back_button": { "size": ["100%","100%"], "$back_button_pressed_button_name": "button.menu_exit" }

$back_button_pressed_button_name 按钮映射事件

值：字符串/映射事件
值等于上面的to_button_id（按下就可以触发这个按钮事件）

修改外观与个性化按钮类似，这里不多一一介绍


使按钮跟随鼠标指针

你可以利用这个特性来做出一些有趣的东西
不过用处不大

follows_cursor 是否跟随准心

值：布尔值
值为true时即可跟随鼠标（触屏可以使用）
按钮控件跟随准心时，可无视offset等改变位置的属性，但仍然会被剪辑面板剪辑


焦点

焦点用于方便选中其他控件，仅游戏手柄和键盘能使用焦点

focus_enabled 焦点是否启用

值：布尔值
是否允许控件使用焦点选择

focus_identifier 焦点标识符

值：字符串
用于定义该控件的焦点
焦点标识符不要重复定义

use_last_focus 使用上一个焦点

值：布尔值
允许保留上一个焦点

focus_container 容器焦点是否启用

值：布尔值
一般用于选中panel中的内容

focus_magnet_enabled 磁铁焦点是否启用

值：布尔值

focus_wrap_enabled 包裹焦点是否启用

值：布尔值

focus_change_down 选中下方的焦点

值：字符串
按控制器或键盘的"↓"时，选中有目标焦点标识符的控件
例如: 值为test时，按"↓"后选择"focus_identifier"为test的控件

如果不要向下移动焦点，值为FOCUS_OVERRIDE_STOP

focus_change_up 选中上方的焦点

值：字符串
按控制器或键盘的"↑"时，选中有目标焦点标识符的控件
例如: 值为test时，按"↑"后选择"focus_identifier"为test的控件

如果不要向上移动焦点，值为FOCUS_OVERRIDE_STOP

focus_change_left 选中左边的焦点

值：字符串
按控制器或键盘的"←"时，选中有目标焦点标识符的控件
例如: 值为test时，按"←"后选择"focus_identifier"为test的控件

如果不要向左移动焦点，值为FOCUS_OVERRIDE_STOP

focus_change_right 选中右边的焦点

值：字符串
按控制器或键盘的"→"时，选中有目标焦点标识符的控件
例如: 值为test时，按"→"后选择"focus_identifier"为test的控件

如果不要向右移动焦点，值为FOCUS_OVERRIDE_STOP

focus_navigation_mode_down 选中选项组下面的焦点

值：字符串/列举
值为"none"时，禁止选择下面焦点
值为"contained"时，选择容器内下面的焦点
值为"custom"时，选择容器外部下面的焦点

focus_navigation_mode_up 选中选项组上面的焦点

值：字符串/列举
值为"none"时，禁止选择上面焦点
值为"contained"时，选择容器内上面的焦点
值为"custom"时，选择容器外部上面的焦点

focus_navigation_mode_left 选中选项组左边的焦点

值：字符串/列举
值为"none"时，禁止选择左边焦点
值为"contained"时，选择容器内左边的焦点
值为"custom"时，选择容器外部左边的焦点

focus_navigation_mode_right 选中选项组右边的焦点

值：字符串/列举
值为"none"时，禁止选择右边焦点
值为"contained"时，选择容器内右边的焦点
值为"custom"时，选择容器外部右边的焦点

focus_container_custom_(up/down/left/right) 选中选项组（上/下/左/右）的外部焦点

值：数组
本属性名包含四个属性，属性名以括号方式列举属性
属性示例: 
"focus_container_custom_right": [
  {
    "other_focus_container_name": "控件名",. .   "focus_id_inside": "内部焦点名"
  }
]

代码示例:

RP/ui/start_screen.json { "test": { "type": "panel", "focus_container": true }, "focus_panel": { "type": "input_panel", "focus_container_custom_right": [ { "other_focus_container_name": "test", "focus_id_inside": "none" } ] } }

其他属性

还有一些属性没有讲，但是他们并不重要！
不过他们仍然为控件做出一些改动

draggable 允许拖动按钮

值：字符串/列举
值为"vertical"时，仅允许竖直方向拖动
值为"horizontal"时，仅允许水平方向拖动
值为"both"时，可任意拖动
拖动时仍受"clips_children"影响
不过，松开手放开按钮时会触发按钮映射事件

always_handle_pointer 总是处理指针位置

值：布尔值

consume_hover_events 启用悬停事件

值：布尔值

tts_name 按下时所朗读的文字

值：字符串
开启"屏幕阅读"有效

tts_control_header 按下时所朗读的头部

值：字符串
开启"屏幕阅读"有效

tts_section_header 按下时所朗读的头部选项

值：字符串
开启"屏幕阅读"有效

RP/ui/start_screen.json "content": { // 这是按钮样式的按钮 "type": "image", "texture": "textures/items/apple" }, "content_hover": { // 这是按钮样式的按钮 "type": "image", "texture": "textures/items/diamond" }, "content_pressed": { // 这是按钮样式的按钮 "type": "image", "texture": "textures/items/bamboo" }, "button@common.button": { //这个就是按钮控件 "size": [100,20],// 这行是按钮的大小 // 下面是按下后的事件 "$pressed_button_name": "button.menu_settings", // 定义按钮外观 "controls": [ { // 此处的content请换成你需要引用的控件 // 引用的控件将作为默认状态显示 "default@content": {} }, { // 此处的content_hover请换成你需要引用的控件 // 引用的控件将作为悬停状态显示 "hover@content_hover": {} }, { // 此处的content_pressed请换成你需要引用的控件 // 引用的控件将作为按下状态显示 "pressed@content_pressed": {} } ] }

对于了解继承变量的玩家，可以在按钮控件的controls引用同一个控件。这样会省下几个控件
但是不建议新手这么做！
例如：

RP/ui/start_screen.json "content": { // 这是按钮样式的按钮 "type": "image", "texture": "$image" }, "button@common.button": { //这个就是按钮控件 "size": [100,20],// 这行是按钮的大小 // 下面是按下后的事件 "$pressed_button_name": "button.menu_settings", // 定义按钮外观 "controls": [ // 此处的content_hover请换成你需要引用的控件 // 引用的控件将作为默认状态显示 "default@content": {"$image": "textures/items/apple"} }, { // 此处的content_hover请换成你需要引用的控件 // 引用的控件将作为悬停状态显示 "hover@content": {"$image": "textures/items/diamond"} }, { // 此处的content_hover请换成你需要引用的控件 // 引用的控件将作为按下状态显示 "pressed@content": {"$image": "textures/items/bamboo"} } ] }
